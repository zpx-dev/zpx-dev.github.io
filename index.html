zpx

A reverse proxy written in Zig.
HTTP/2 frontends · h2c multiplexed upstreams · kTLS offload · ACME · hot reload · Docker discovery

Zig 0.16-dev HTTP/2 + h2c TLS 1.3 + kTLS ACME MIT License

Quickstart

# Build
zig build -Doptimize=ReleaseFast

# Create config
cat > zpx.toml << 'EOF'
[server]
bind = "0.0.0.0:8080"

[[service]]
name = "app"
backends = ["http://localhost:3000"]

[[route]]
path = "/"
service = "app"
EOF

# Run
./zig-out/bin/zpx zpx.toml

zpx is now proxying traffic on :8080 to your backend on :3000.

Add TLS

[server]
bind = "127.0.0.1:8080"
tls_cert = "/path/cert.pem"
tls_key = "/path/key.pem"

[[listener]]
name = "https"
bind = "0.0.0.0:443"

[[listener]]
name = "http"
bind = "0.0.0.0:80"
tls = false

Listeners handle the public-facing ports. TLS listeners terminate TLS and negotiate HTTP/2 via ALPN. Plain listeners (tls = false) handle HTTP/1.1 directly — useful for HTTP-to-HTTPS redirects or ACME HTTP-01 challenges.

Validate config

zpx validate zpx.toml

Configuration

zpx uses TOML. Pass the config path as the first argument (defaults to zpx.toml).

Config is split into services (backend pools) and routes (matching + middleware). Multiple routes can share a service.

Server

[server]
bind = "0.0.0.0:8080"            # Internal HTTP listen address (IPv6: "[::]:8080")
workers = 4                       # CPU-pinned worker threads (default: CPU count, max 32)
tls_cert = "/path/cert.pem"       # TLS certificate (optional if using ACME)
tls_key = "/path/key.pem"         # TLS private key (optional if using ACME)
proxy_timeout = 30                # Upstream timeout in seconds (1-3600, default: 30)
max_request_body = 10485760       # Max request body bytes (default: 10 MB, max: 1 GiB)
max_response_body = 67108864      # Max response body bytes (default: 64 MB, max: 1 GiB)
send_strategy = "msg_more"        # kTLS send path: "msg_more" | "cork" (default: msg_more)
admin_bind = "127.0.0.1:9090"    # Admin dashboard address (optional)
admin_token = "secret"            # Bearer token for dashboard auth
admin_insecure = false            # Skip auth for trusted networks

When admin_bind is set, you must provide either admin_token or admin_insecure = true.

Listeners

Listeners bind public-facing ports. Each listener is either TLS (default) or plain.

[[listener]]
name = "https"                    # Unique name (required)
bind = "0.0.0.0:443"             # Listen address (required)
tls = true                        # TLS termination (default: true)

[[listener]]
name = "http"
bind = "0.0.0.0:80"
tls = false                       # Plain HTTP (no TLS)

TLS listeners terminate TLS 1.3 with ALPN negotiation. HTTP/2 connections use nghttp2 framing; HTTP/1.1 connections use a zero-copy proxy path. Both go through the same CPU-pinned worker pool. Plain listeners handle HTTP/1.1 directly without TLS — useful for ACME HTTP-01 challenges and redirects.

tls_cert and tls_key are required when any listener has tls = true (unless acme_domain is set).

Routes can be restricted to a specific listener with listener = "name".

Services

A service defines a backend pool with balancing, health checks, and circuit breaker settings.

[[service]]
name = "api"                     # Unique name (required)
backends = [                     # One or more upstream URLs
  "http://10.0.0.1:3000",
  "http://10.0.0.2:3000",
  "h2c://10.0.0.3:3000",        # HTTP/2 cleartext (multiplexed)
]

# Load balancing
balance = "round_robin"          # round_robin | least_conn | least_latency
                                 # consistent_hash | peak_ewma | p2c

# Health checks
health_path = "/healthz"         # Endpoint to poll (default: /healthz)
health_interval = 10             # Poll interval in seconds (1-3600)

# Circuit breaker
circuit_break_threshold = 5      # Consecutive failures before open (default: 5)
circuit_break_duration = 30      # Seconds before half-open retry (default: 30)
max_conns = 0                    # Max concurrent connections per backend (0 = unlimited)

# IP allowlist
ip_allowlist = "10.0.0.0/8, 192.168.0.0/16"  # Comma-separated CIDRs
ip_allowlist_deny = "403"        # "403" or "drop" (TCP RST)

# gRPC streaming
protocol = "http"                # "http" | "grpc"
grpc_bind = "0.0.0.0:9000"      # Dedicated gRPC listener (required when protocol = "grpc")

Backend URL schemes

  • http:// — HTTP/1.1 cleartext
  • https:// — HTTP/1.1 over TLS
  • h2c:// — HTTP/2 cleartext with multiplexed streams, streaming flow control, and GOAWAY retry

Routes

A route matches requests by host and/or path and forwards them to a service. Middleware is configured per-route.

[[route]]
host = "api.example.com"         # Match by Host header (optional)
path = "/v1"                     # Match by path prefix (optional)
service = "api"                  # Target service (required)
listener = "https"               # Restrict to named listener (optional)
weight = 10                      # Explicit priority (higher wins, default: 0)

# Logging
log = true                       # Enable request logging
log_json = true                  # JSON output format

# Security
rate_limit_rps = 100             # Max requests/sec per IP (0 = unlimited)
ip_allowlist = "10.0.0.0/8"     # CIDR access control
ip_allowlist_deny = "403"        # "403" or "drop"
hsts = true                      # Strict-Transport-Security header
hsts_max_age = 63072000          # Max-age in seconds (default: 2 years)
hsts_subdomains = true           # includeSubDomains directive
hsts_preload = false             # preload (requires subdomains + max_age >= 1 year)

# Headers
cors_origins = "*"               # CORS allowed origins
add_header = "X-Proxy"           # Custom response header name
add_header_value = "zpx"         # Custom response header value

# Content
compress = true                  # Response compression (br/zstd/gzip)

# WebSocket
ws_allowed_origins = "https://app.example.com"  # Restrict WS upgrade origins

# Mirroring
mirror = "http://shadow:3000"    # Mirror traffic URL (optional)
mirror_percent = 50              # Percentage to mirror (default: 100)

Mirroring: Fire-and-forget — the mirror request is sent in a detached thread and does not affect primary request latency. H2 always mirrors the full request body. H1 mirrors the body only when Content-Length is 1 MB or less; larger or chunked bodies are mirrored without body content.

Routing priority

  1. Explicit weight (higher wins)
  2. Specificity score: host match (2 pts) + path match (1 pt)
  3. First match in config order

Host and path values prefixed with ~ use POSIX ERE regex: host = "~^.*\.example\.com$".

Config Includes

include = "conf.d/*.toml"

Included files can contain [[service]] and [[route]] sections. Glob patterns support * wildcards. Paths are relative to the main config directory. Files in config.d/*.toml are always loaded if the directory exists.

ACME — Automatic TLS

zpx integrates with acme.sh to provision and renew TLS certificates from Let's Encrypt / ZeroSSL.

Prerequisites

curl https://get.acme.sh | sh

HTTP-01 Challenge

For single-domain certs. zpx serves the challenge files automatically.

[server]
bind = "0.0.0.0:8080"
acme_domain = ["example.com"]
acme_email = "you@example.com"

[[listener]]
name = "https"
bind = "0.0.0.0:443"

[[listener]]
name = "http"
bind = "0.0.0.0:80"
tls = false

DNS-01 Challenge (Wildcard)

For wildcard certificates. Supports 90+ DNS providers.

[server]
bind = "127.0.0.1:8080"
acme_domain = ["*.example.com", "example.com"]
acme_email = "you@example.com"
acme_dns_provider = "dns_cf"
acme_dns_credentials = [
  "CF_Key=your_api_key_here",
  "CF_Email=you@example.com",
]
acme_data_dir = "/etc/zpx/acme"

[[listener]]
name = "https"
bind = "0.0.0.0:443"

How It Works

  1. On startup — zpx runs acme.sh --issue to obtain certs, then --install-cert to copy them to {acme_data_dir}/certs/
  2. Every 12 hours — a background thread runs acme.sh --renew and atomically hot-reloads the TLS certificate if renewed
  3. Zero downtime — existing connections keep the old cert, new connections get the fresh one

ACME Config Reference

FieldDefaultDescription
acme_domain[]Domains to provision (triggers ACME when non-empty)
acme_email—Account email for the ACME provider
acme_data_dir/etc/zpx/acmeWhere acme.sh stores state and certs are installed
acme_dns_provider—DNS provider for DNS-01 (e.g. dns_cf, dns_aws, dns_gd)
acme_dns_credentials[]Provider credentials as KEY=value pairs

Docker Discovery

zpx auto-discovers containers via Docker socket labels.

Container Labels

Labels use a namespaced format: zpx.service.<name>.<field> and zpx.route.<name>.<field>. A single container can expose multiple services and routes.

Minimal

zpx infers the backend from the container's IP and exposed port:

# docker-compose.yml
services:
  myapp:
    image: myapp:latest
    ports:
      - "3000:3000"
    labels:
      zpx.enable: "true"
      zpx.route.myapp.host: "`example.com`"

Full example

services:
  myapp:
    image: myapp:latest
    labels:
      zpx.enable: "true"
      zpx.service.api.port: "8080"
      zpx.service.admin.url: "`http://10.0.0.5:9090`"
      zpx.service.api.health_path: "/health"
      zpx.route.main.host: "`example.com`"
      zpx.route.main.path: "`/api`"
      zpx.route.main.service: "api"
      zpx.route.wild.host: "`~^.*\\.example\\.com$`"

Service labels

zpx.service.<name>.*

FieldDescription
portContainer port (builds http://{container_ip}:{port})
urlExplicit backend URL
health_pathHealth check path (default: /healthz)

When neither port nor url is set, zpx uses the container's IP and first exposed TCP port.

Route labels

zpx.route.<name>.*

FieldDescription
hostHost matching — exact or regex with ~ prefix
pathPath matching — prefix or regex with ~ prefix
serviceTarget service name (defaults to the route's own name)

Wrap values in backticks to prevent YAML escaping issues. Prefix with ~ for POSIX ERE regex.

Design

  • Unified worker pool — N CPU-pinned threads (N = CPU count, max 32), each with its own epoll fd. Both HTTP/2 and HTTP/1.1 connections go through the same pool. Epoll uses 4-way pointer tagging (bits 0-1) to distinguish H2 clients, H1 clients, h2c upstream, and H1 upstream sockets.
  • Named listeners — each [[listener]] binds a port, either TLS or plain. TLS listeners negotiate ALPN (h2 or http/1.1). Plain listeners accept HTTP/1.1 directly. Routes can target a specific listener by name.
  • kTLS offload — after TLS handshake, cipher keys are pushed to the kernel via setsockopt. Subsequent encrypt/decrypt happens in kernel space, avoiding userspace copies.
  • h2c multiplexed upstream — h2c:// backends use HTTP/2 cleartext with streaming flow control, 1 MB connection windows, and GOAWAY graceful migration (REFUSED_STREAM retries on fresh connections).
  • Service/route separation — services define backend pools (balancing, health, circuit breakers). Routes define matching and middleware. Multiple routes can share a service.
  • Atomic cert swap — TLS certificate reload is lock-free. AuthRef is ref-counted; the accept loop atomically loads the current pointer for each new connection.
  • Ref-counted hot reload — SIGHUP swaps the entire routing table atomically. Old state is freed after all in-flight requests complete.
  • Specificity routing — routes scored by host (2 pts) + path (1 pt), with explicit weight override. POSIX ERE regex supported.
  • Content negotiation — compression negotiates Accept-Encoding with q-value parsing, preferring brotli > zstd > gzip.
  • Fire-and-forget mirroring — request body pre-buffered (up to 1 MB) and sent to a secondary URL in a detached thread with percentage-based sampling.

# Debug build
zig build

zig build -Doptimize=ReleaseFast

zig build -Doptimize=ReleaseFast -Dtarget=x86_64-linux-musl
